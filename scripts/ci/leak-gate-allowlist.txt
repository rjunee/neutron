# scripts/ci/leak-gate-allowlist.txt
#
# The ONLY exception mechanism for the PUBLIC leak-gate subset
# (scripts/ci/leak-gate.sh). Format, one entry per line:
#
#   <path-glob>:<rule-id>
#
# A finding is suppressed iff its file path matches <path-glob> AND its rule-id
# matches <rule-id>. Rule ids are the bracketed names in leak-gate.sh output
# (tenant-code, secret-file, …). Keep this file SHORT — every entry is a
# reviewable, committed exception. This is the public-side counterpart to the
# private scripts/sprint-c/leak-gate-allowlist.txt; it carries ONLY the entries
# that are needed for the Tier-2/Tier-3 subset (no Tier-1 PII entries exist here
# because the public subset runs no Tier-1 PII rules).

# Migration 0074 backfills onboarding_state rows persisted at the old
# `tenant_provisioned` onboarding phase value to `instance_provisioned`. Its
# WHERE clause MUST name the old phase string to find the rows to rename, so the
# file unavoidably carries the token — the one legitimate, permanent reference in
# the Open tree. Data migrations are immutable history. Permanent tenant-code FP.
migrations/0074_rename_tenant_provisioned_phase.sql:tenant-code

# .env.example is the zero-PII self-host env TEMPLATE (every value is a default/
# empty placeholder — no real secrets). Its name unavoidably collides with the
# secret-file `.env*` pattern. Carried into Open on purpose (the quickstart's
# `cp .env.example .env` step needs it). Permanent template false-positive.
.env.example:secret-file
