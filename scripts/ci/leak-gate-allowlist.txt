# scripts/ci/leak-gate-allowlist.txt

# The ONLY exception mechanism for the PUBLIC leak-gate subset
# (scripts/ci/leak-gate.sh). Format, one entry per line:
#
#   <path-glob>:<rule-id>
#
# A finding is suppressed iff its file path matches <path-glob> AND its rule-id
# matches <rule-id>. Rule ids are the bracketed names in leak-gate.sh output
# (tenant-code, secret-file, …). Keep this file SHORT — every entry is a
# reviewable, committed exception. This is the public-side counterpart to the
# private scripts/sprint-c/leak-gate-allowlist.txt; it carries ONLY the entries
# that are needed for the Tier-2/Tier-3 subset (no Tier-1 PII entries exist here
# because the public subset runs no Tier-1 PII rules).

# tasks/history-import-seeder.ts embeds the literal `tenant:` as a FROZEN
# hash-seed / domain-separation label fed into the SHA-256 that mints every
# `hi_<sha256>` history-import task id. It predates the tenant→owner purge and is
# NOT live multi-tenant surface — but changing it would re-mint every existing
# task id and break the idempotent re-seed guard, so it is locked. Before unit G7
# the field separator was a RAW 0x00 NUL byte that made grep classify the whole
# file as binary and skip it, hiding this token entirely; G7 replaced the NUL
# with the byte-identical `\x00` escape (hash-stability proven by test) so the
# gate now SEES the file — hence this exact-path exemption for the retired-vocab
# rules it now legitimately matches (the seed literal + its explanatory comment).
# PII / hosted-domain / secret rules stay ARMED on the file.
