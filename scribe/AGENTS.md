# AGENTS.md — scribe

This module owns the per-project scribe budget + entity-extraction pipeline. The prompt itself is lifted from Nova's `prompts/scribe.md` (parameterized for the instance home in Sprint 2). Writes to `<instance-home>/entities/`; per-project budget tracked in the `scribe_budget` table of the project database. Implementation lands in P1.

It must NOT cross instance boundaries or share budget state across instances. Budget reset is per-day, per-instance.

**Compiled-truth is append-only BY DEFAULT — with one flag-gated exception (RB4).** The base rule stands: a sparse chat turn never overwrites or retracts a richer existing page; new facts land in the append-only timeline, and only genuinely-new relationship sentences are appended to compiled-truth (see the `write-to-gbrain.ts` module header). The SOLE exception is RB4 temporal invalidation (belief evolution), gated behind the shared `NEUTRON_PERFECT_RECALL` flag: when a relation carries an explicit `supersedes` marker AND the flag is on, the superseded `(predicate, prior-object)` SENTENCE is removed from compiled-truth so the current truth (and the gbrain edge) reflect the change. This removal is deliberately CONSERVATIVE: a sentence is dropped only when it asserts nothing but superseded relations; a COMPOUND sentence that also carries a still-current relation or descriptive prose is kept BYTE-FOR-BYTE (never deleted, never reconstructed from templates) — hand-authored prose is never mangled, at the cost of leaving that rare embedded superseded relation in place. Scribe's own prose is one relation per sentence, so real chat-time supersessions always hit the clean path. **Scope: OBJECT REPLACEMENT ONLY** — a subject gains a NEW object for the SAME predicate, retiring the prior object (`works_at OldCo → works_at NewCo`). RB4 does NOT model an entity RENAME (the entity's own identity changing) or an ended affiliation with NO replacement (`Jane left OldCo` with no new employer); those key on nothing to retire and are out of scope — such turns simply accrete normally, no supersession applied.

**Inherent gbrain KG bound (not RB4-specific).** The graph holds ONE edge per (subject, object) pair — the STRONGEST predicate (the `FOUNDED ≻ INVESTED ≻ ADVISES ≻ WORKS_AT ≻ …` cascade). RB4 retires a superseded relation when it is the object's LIVE edge (the common, scribe-authored, one-relation-per-sentence case — always correct). Where a single object carries MULTIPLE predicates (e.g. the subject both advises and works-at the same company), only the strongest is an edge; a weaker superseded predicate to that object has no distinct edge to retire, so its prose may remain while the graph already reflects the stronger current relationship. This is the pre-existing KG collapse model, unchanged by RB4.

**No superseded fact is ever silently dropped — the append-only timeline is the durable history.** The superseding turn ALWAYS records a dated `superseded <pred>: prior → new` note at the invalidation time, so the retired assertion survives in the timeline even in the flag-off→on transition (a `works_at oldco` written while the flag was off carries no relation note of its own, and its original date is unrecoverable — the invalidation-time note is then the record that the fact was ever true). Additionally, when the original assertion was itself written under the flag it also carries its own dated `<pred> <obj>` row at its original observation time. Either way the fact is preserved with a date; only prior *timeline* rows are never rewritten. Flag OFF → pure accretion, exactly as before. No other path may rewrite existing entity prose.
