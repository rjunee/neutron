# AGENTS.md — scribe

This module owns the per-project scribe budget + entity-extraction pipeline. The prompt itself is lifted from Nova's `prompts/scribe.md` (parameterized for the instance home in Sprint 2). Writes to `<instance-home>/entities/`; per-project budget tracked in the `scribe_budget` table of the project database. Implementation lands in P1.

It must NOT cross instance boundaries or share budget state across instances. Budget reset is per-day, per-instance.

**Compiled-truth is append-only BY DEFAULT — with one flag-gated exception (RB4).** The base rule stands: a sparse chat turn never overwrites or retracts a richer existing page; new facts land in the append-only timeline, and only genuinely-new relationship sentences are appended to compiled-truth (see the `write-to-gbrain.ts` module header). The SOLE exception is RB4 temporal invalidation (belief evolution), gated behind the shared `NEUTRON_PERFECT_RECALL` flag: when a relation carries an explicit `supersedes` marker AND the flag is on, the superseded `(predicate, prior-object)` SENTENCE is removed from compiled-truth so the current truth (and the gbrain edge) reflect the change. This removal is deliberately CONSERVATIVE: a sentence is dropped only when it is PURELY a single generated relationship assertion for a superseded target (exactly one graph relation, normalising — case-insensitively, matching the extractor — to the rendered `RELATION_SENTENCE` template). Anything else — a COMPOUND sentence asserting more than one relation, or a single relation wrapped in descriptive prose — is kept BYTE-FOR-BYTE (never deleted, never reconstructed) — hand-authored prose is never mangled, at the cost of leaving that rare embedded superseded relation in place. Scribe's own prose is one relation per sentence, so real chat-time supersessions always hit the clean path. **Scope: OBJECT REPLACEMENT ONLY** — a subject gains a NEW object for the SAME predicate, retiring the prior object (`works_at OldCo → works_at NewCo`). RB4 does NOT model an entity RENAME (the entity's own identity changing) or an ended affiliation with NO replacement (`Jane left OldCo` with no new employer); those key on nothing to retire and are out of scope — such turns simply accrete normally, no supersession applied.

**Inherent gbrain KG bound (not RB4-specific).** The graph holds ONE edge per (subject, object) pair — the STRONGEST predicate (the `FOUNDED ≻ INVESTED ≻ ADVISES ≻ WORKS_AT ≻ …` cascade). RB4 retires a superseded relation when it is the object's LIVE edge (the common, scribe-authored, one-relation-per-sentence case — always correct). Where a single object carries MULTIPLE predicates (e.g. the subject both advises and works-at the same company), only the strongest is an edge; a weaker superseded predicate to that object has no distinct edge to retire, so its prose may remain while the graph already reflects the stronger current relationship. This is the pre-existing KG collapse model, unchanged by RB4.

**The append-only timeline is the durable history — via ADDITIVE dated rows, not a transition note.** Under the flag, every write records its relation assertions ADDITIVELY as dated `<pred> <obj>` rows (a PURE function of the extraction — so a replay is byte-identical, nothing is ever fabricated, and no free-text timeline is ever parsed back). The current truth is carried by the compiled-truth removal + the gbrain edge invalidation; the before/after beliefs are each their own additive dated row (`works_at oldco` at its original time survives untouched; `works_at newco` lands at the superseding turn's time). There is deliberately NO state-dependent `superseded X → Y` note — that design caused a cascade of fabrication / idempotency / injection / forgery hazards (untrusted model facts + un-authenticated timeline text), all eliminated by the additive-only model. LIMITATION: a relation first recorded while the flag was OFF has no additive row of its own, so in the flag-off→on transition the retired prior isn't NAMED in a note (its original generic entry + entity page remain; the current-truth removal + edge invalidation carry the change). Flag OFF → pure accretion, exactly as before. No other path may rewrite existing entity prose.
