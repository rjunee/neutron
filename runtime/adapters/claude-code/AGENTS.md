# AGENTS.md — runtime/adapters/claude-code

This module owns the Claude Code substrate adapter — `Substrate.start(spec) → SessionHandle`. `tool_resolution: 'internal'` (Claude Code resolves MCP tools server-side inside the spawned `claude` process; `respondToTool` THROWS if called). Composition: `index.ts` is the `createClaudeCodeSubstrateAuto` selector — post the **2026-06-07 S3 rip-replace** it UNCONDITIONALLY maps onto `persistent/persistent-repl-substrate.ts:createPersistentReplSubstrate` (the persistent interactive REPL). The legacy per-turn `claude -p` transport (`cli-transport.ts`), its cancel helper (`cancel.ts`), and the `claude -p` warm-router (`warm-router-process.ts`) were **HARD-DELETED** — there is no flag and no `claude -p` fallback. `api-key-helper.ts` (the slim `makeMaxOAuthSubscriptionLoader`, formerly consumed by `gateway/cores/code-gen-factory.ts`) was itself **HARD-DELETED** in the K5 kill-list sweep once `code-gen-factory.ts` was retired in the Trident-port close-out — the `claude` binary delegates auth to `~/.claude/.credentials.json` and the env vars composers thread in via the spawn env, so nothing in this adapter needs a standalone OAuth loader.

**It MUST spawn the `claude` CLI as the substrate for every LLM dispatch.** `Substrate.start(spec) → SessionHandle` is implemented by hosting ONE persistent interactive `claude` REPL per `(substrate_instance_id, user_id, project_id, credential_identity)` (Bun-native PTY + dev-channel; NO `-p`, NO `--output-format stream-json`) and bridging each turn's `reply()` to one `completion` Event (`runtime/events.ts`). Direct HTTP calls to `https://api.anthropic.com/v1/messages` are FORBIDDEN — they require impersonating the Claude Code system-prompt signature (`"You are Claude Code, Anthropic's official CLI for Claude."`) to pass the Max OAuth gate, which is (a) a ToS risk (potential Anthropic ban) and (b) breaks every time Anthropic rotates the gate. The interactive REPL (1) inherits Claude Code's official auth/header/system-prompt handling, (2) stays exempt from the June-15-2026 `claude -p` metering cap (an interactive Max session is metered as human-interactive), (3) makes Neutron's instance-side LLM dispatch indistinguishable from Nova's interactive agents at the transport layer. **Tool restriction (Codex-r1-P1, SECURITY):** the REPL spawn ALWAYS emits `--tools` (default-deny — `spec.tools:[]` → `--tools ""`), so untrusted-content callers (history-import) can't reach Bash/Read/Write under `--dangerously-skip-permissions`.

It must NOT pool personal Max OAuth tokens between humans (`engineering-plan.md` line 142), populate `dollars` in `completion` events (CC has its own subscription; Neutron is not in the billing path — `engineering-plan.md` line 433), or fingerprint Anthropic env vars in a way that surfaces credential identity into the prompt. Multi-model rotation lives inside the adapter via `model_preference: string[]`; same-user multi-sub credential rotation is fed via env-var overrides at `start()` time (composers like `gateway/realmode-composer/build-import-substrate.ts` thread `ANTHROPIC_API_KEY` / `ANTHROPIC_AUTH_TOKEN` per call so the `claude` binary's own auth resolution picks the per-turn credential).

Multi-turn continuity is delegated to Claude Code's `--session-id` flag — the binary writes the JSONL transcript at `~/.claude/projects/<encoded-cwd>/<session_id>.jsonl` itself. Adapter no longer reads or writes transcripts directly.
